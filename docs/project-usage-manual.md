# 项目使用手册

本文面向访客、普通用户、作者、编辑和管理员，说明产品功能入口、操作流程和权限边界。接口字段请参阅 [`API 接口大全`](api-reference.md)；aaPanel/Linux 部署请参阅 [`生产安全部署规范`](production-security.md) 和 [`aaPanel 完整部署教程`](aapanel-deployment.md)。

> **核验状态**：本文档基于当前源码、视图、前端组件和 Feature 测试整理。文中标为“已确认”的内容是源码可直接确认的行为；涉及域名、HTTPS、邮件投递、Session/Cookie 最终配置、代理、缓存、文件权限和生产环境参数的内容，均需部署后核验。本文档不替代部署文档或站点运营规则。

## 1. 侦察结论与项目定位

### 1.1 侦察范围

本次按 `01-recon` 的思路完成了以下顺序：

1. 从 `routes/web.php` 和 `routes/api.php` 盘点公开、认证、作者和管理员入口。
2. 沿路由追踪 `app/Http/Controllers`，核对认证、数据校验、状态变化和授权判断。
3. 阅读 `resources/views`，确认用户实际看到的导航、表单和页面入口。
4. 阅读 `resources/js/vue`，确认阅读器、评分、语言、主题、私信和群组的交互。
5. 对照 `tests/Feature` 的认证、角色、公开内容、互动、投稿、通信、本地化和主题测试，确认权限边界与异常行为。

### 1.2 项目定位

这是一个基于 Laravel 13 的在线阅读与作品管理平台：访客可以浏览已发布作品、分类和章节；登录用户可以建立个人阅读记录、收藏、评分和交流；作者可以投稿并管理自己的作品；编辑可以访问其授权的内容工作区和投稿详情，管理员负责投稿审核、站点配置、分类、内容和审计日志。投稿批准/拒绝操作由管理员接口提供，编辑不通过普通投稿接口查看全量投稿。

项目的主要入口如下：

| 场景 | 入口 |
| --- | --- |
| 首页、作品库、分类、关于、阅读指南、联系 | `/`、`/novels`、`/categories`、`/about`、`/reading-guide`、`/contact` |
| 作品详情与阅读 | `/novels/{slug}`、`/novels/{slug}/chapters/{number}` |
| 个人中心 | `/dashboard` |
| 私信与群组 | `/messages`、`/groups`，也可从个人中心的社交分组进入 |
| 作者工作区 | `/author/novels` 及其作品编辑、章节入口 |
| 投稿 | 个人中心的“投稿”区域，或 `/submissions`、`/author/submissions` |
| 管理员后台 | `/admin` 及 `/admin/settings`、`/admin/categories`、`/admin/novels`、`/admin/submissions`、`/admin/audit-logs` |

公开页面只展示 `published` 状态的作品和章节。草稿、归档或不存在的内容不能通过公开阅读入口访问；登录后阅读已发布章节时，系统会记录当前用户的阅读进度。

## 2. 角色与权限模型

### 2.1 角色说明

| 角色 | 可做什么 | 明确不能做什么 |
| --- | --- | --- |
| `user` | 浏览、阅读、收藏、评分、查看阅读记录、私信、创建或参与群组、修改自己的账户设置 | 没有投稿入口，不能访问作者工作区、编辑全部作品的能力或管理员后台 |
| `author` | 拥有 `user` 的能力；投稿；查看和管理自己名下的作品、封面、状态和章节 | 不能访问管理员后台；不能管理其他作者的作品 |
| `editor` | 拥有阅读和交流能力；访问投稿流程；管理全部作者作品和章节；参与内容工作流 | 不能访问管理员后台的站点设置、分类、全局作品管理和审计日志 |
| `admin` | 拥有全部前台、作者和编辑能力；访问管理员后台，管理站点、分类、所有作品、章节、投稿和投稿审计日志 | 应按最小权限使用；普通注册流程不能自行注册为 `admin` |

注册表单只接受 `user` 或 `author`，省略角色时默认为 `user`。`editor` 和 `admin` 不可通过公开注册获得，必须由既有管理流程或部署初始化授予。未知角色不会绕过授权检查。

### 2.2 认证前置条件

- `/dashboard`、私信、群组、收藏、评分、阅读记录要求登录。
- 投稿、作者工作区和管理员后台同时要求 `auth`、`email.required`；当站点启用邮箱验证时，未验证邮箱的账号会被拒绝进入这些受保护区域。
- `email_verification_required` 由站点配置控制。关闭时，未验证用户仍可进入不受邮箱验证中间件保护的功能；开启后，需先完成验证。
- 角色判断发生在路由中间件和控制器授权中，不能仅靠隐藏导航来获得权限。

## 3. 注册、登录、邮箱验证与二次验证

### 3.1 注册

1. 打开 `/register`，填写名称、邮箱、密码和确认密码。
2. 如需作者身份，在允许的角色选项中选择 `author`；不要提交 `editor`、`admin` 或未知角色。
3. 注册成功后会自动登录并进入 `/dashboard`。
4. 如果站点启用了邮箱验证，系统会发送验证通知；邮件发送暂时失败不会撤销已创建账号，但应通过“重新发送验证邮件”补发。

注册接口受 `throttle:register` 限流。邮箱必须唯一，密码必须通过 Laravel 默认密码规则。

### 3.2 登录、退出与找回密码

- 在 `/login` 使用邮箱和密码登录，可选择记住登录状态。
- 登录失败会返回凭据错误；登录接口受 `throttle:login` 限流。
- 成功登录后通常跳转到原目标页面或 `/dashboard`。
- 退出通过页面的退出操作提交 `POST /logout`；退出后回到首页并清理会话和缓存控制头。
- 忘记密码从 `/forgot-password` 请求邮件，再通过 `/reset-password/{token}` 设置新密码。无效或过期令牌会进入密码重置失败入口；重置成功后应重新登录。

### 3.3 邮箱验证

- 注册后按照邮件中的签名链接访问 `/auth/email/verify/{id}/{hash}`。
- 链接验证签名和邮箱哈希；验证成功后才会标记 `email_verified_at`。
- 已验证账号再次访问会收到“已验证”提示。
- 登录后如果没有收到邮件，可以调用验证通知入口重新发送；该入口受每分钟 6 次的限流保护。
- 若出现“验证失败”，先检查 SMTP 配置、邮件地址、链接是否过期，以及站点是否启用了邮箱验证要求。

### 3.4 二次验证（TOTP）

二次验证分为“账号设置”和“登录挑战”两步：

1. 登录后打开账户设置中的二次验证页面 `/auth/two-factor`。
2. 点击启用，获取二维码或密钥；在身份验证器中添加账号。
3. 使用当前 TOTP 验证码确认启用。只生成设置信息但未确认时，不算已启用。
4. 下次登录在密码通过后会跳转到 `/two-factor/challenge`，填写 TOTP 验证码，或填写恢复码完成登录。
5. 恢复码只能按服务端实现使用；请在启用二次验证时妥善保存，不要公开提交。
6. 禁用二次验证需要当前密码或有效 TOTP 验证码。

挑战具有有效期，默认配置由 `yuejing.two_factor.challenge_lifetime` 控制；错误次数达到 `yuejing.two_factor.max_attempts` 后挑战会锁定并需要重新发起登录。挑战接口同时受 `throttle:two-factor` 限流。遇到挑战过期、锁定或没有待处理挑战时，回到登录页重新登录。

## 4. 阅读、收藏、评分与阅读记录

### 4.1 发现与阅读

1. 首页查看推荐、最新和排行内容。
2. 在 `/novels` 使用关键词、分类和排序筛选作品；支持热门、最新等排序，并带分页。
3. 打开作品详情查看简介、作者、分类、已发布章节、收藏数和评分统计。
4. 进入章节阅读页开始阅读。公开阅读地址支持 `/novels/{slug}/chapters/{number}` 和兼容入口 `/chapters/{slug}/{number}`。
5. 未登录可以阅读公开内容；登录后阅读会写入或更新自己的 `reading_records`，个人中心会显示最近阅读和进度。

未发布作品或章节不会被公开控制器返回。作品不存在、章节不属于该作品或章节尚未发布时，应从作品详情重新选择可用章节，而不是继续猜测 URL。

### 4.2 收藏

- 登录后在作品详情点击收藏；再次操作可取消收藏。
- 收藏和取消收藏只针对已发布作品；不存在或未发布作品返回 404。
- 在个人中心导航的“收藏”或 `/account/favorites` 查看收藏列表。列表会过滤掉当前已不再发布的作品。

### 4.3 评分与评分附带文字

- 登录后在作品详情填写总评分。系统通过 `RatingScale` 校验分段刻度：`1.0–5.0`、`6.0–7.0`、`8.0–9.0` 和 `9.1–9.9`（一位小数）；不在这些区间内的值会被拒绝，可附带评分文字。
- 还可以填写剧情、文笔、人物、原创性四个维度，每项为 1 至 10 的整数。
- 已有有效评分时不能直接覆盖。必须先使用“撤回评分”，再提交新评分；否则返回冲突错误。
- 评分只允许针对已发布作品；作品下线后评分和撤回入口也会被拒绝。
- 公开评分及评分附带文字通过作品详情和 `/api/novels/{slug}/reviews` 提供。这里的“review”是评分记录中的可选文字，不是独立的评论发布、回复或审核系统。后台虽然有 `allow_comments` 配置字段，但当前评分控制器没有用它阻断评分或评分附带文字；部署和运营时不要把该字段当成已生效的评论权限开关。

### 4.4 阅读器设置

阅读章节时，`ReaderControls.vue` 提供：

- 减小或增大正文文字；当前范围为 14 至 26 像素，默认 18 像素。
- 切换当前阅读页的夜间模式。

字体大小和阅读页夜间状态写入浏览器 `localStorage`，键名分别为 `yuejing-reader-size` 和 `yuejing-reader-night`。清除浏览器存储后会回到默认值；这与站点全局主题不是同一个设置。

## 5. 私信与群组

### 5.1 私信

1. 从个人中心的“社交”分组进入私信，或直接打开 `/messages`。
2. 在用户搜索框按名称搜索其他用户，选择对象后建立或打开会话。
3. 在会话中查看历史消息、发送消息；消息正文去除首尾空白，最长 5000 字。
4. 进入会话后可标记对方消息为已读；前端使用同源 SSE 流接收新消息。该流按请求的超时参数结束，最长 30 秒，不是永久连接。

安全边界：私信要求登录；用户不能给自己发消息；只有会话两端成员可以查看会话、标记已读或连接消息流。直接修改会话 ID 访问他人的会话会返回 403。私信接口使用浏览器 Session Cookie 和 CSRF，而不是 Bearer Token。

### 5.2 群组

1. 从个人中心的“社交”分组进入群组，或直接打开 `/groups`。
2. 填写群组名称，可在创建时选择成员；创建者自动加入并成为 `owner`。
3. 选择自己所属的群组查看成员和消息；成员可以发言、标记已读并通过有超时上限的 SSE 流接收更新。群组流最长 10 秒，结束后应按客户端策略重新连接。
4. `owner` 或群组成员角色为 `admin` 的成员可以添加成员。
5. 只有 `owner` 可以移除成员，不能移除群组创建者本人。

群组权限是群组内部的 `owner`、`admin`、`member`，不要与站点的 `user`、`author`、`editor`、`admin` 角色混淆。非成员查看群组、发送消息或标记已读都会被拒绝；用户只能在自己的群组列表中看到所属群组。

## 6. 投稿与作品、章节管理

### 6.1 投稿入口与提交内容

作者、编辑和管理员从个人中心的“投稿”进入。普通 `user` 不显示投稿入口，直接访问也会被拒绝。首次投稿必须提供简介；每次投稿都必须提供标题、正文和封面。

正文有两种互斥来源：

- 在 Markdown 编辑器或文本框中输入正文，可选择 `markdown` 或 `text`。
- 上传稿件文件；当前解析器会读取文件内容和格式，只保存解析后的内容，不保存原始上传文件。

不要同时提交编辑器正文和稿件文件，否则会触发来源冲突校验。支持的封面来源是图片上传或合法的 `cover_url`；文件大小和图片格式按控制器校验规则执行。

### 6.2 查看投稿状态

个人中心投稿区域会显示投稿历史和状态，常见状态为 `pending`、`approved`、`rejected`。作者可以查看自己的投稿；编辑和管理员可以查看审核范围内的投稿详情。投稿建立时会产生 `submission.created` 审计记录。

### 6.3 审核后的结果

- 管理员在 `/admin/submissions` 审核，状态只能选择批准或拒绝，并可填写审核备注。
- 批准新投稿时，系统会创建或关联作品，并将投稿正文创建为第一章；作品和第一章会成为已发布内容。
- 拒绝投稿不会创建新的作品；投稿会保留审核人、审核时间、状态和备注。
- 已处理的投稿不能重复审核，会返回冲突错误。

### 6.4 作品管理

作者工作区 `/author/novels` 展示作品列表。作者只能管理自己名下的作品；编辑和管理员可以管理全部作者作品。作品编辑支持：

- 修改标题、简介和分类。
- 替换封面或更新封面 URL。
- 在 `draft`、`published`、`archived` 间调整状态。

作品状态改为 `published` 时会设置发布时间；改为 `draft` 时会清除发布时间。草稿和归档作品不应被当作公开阅读内容。

### 6.5 章节管理

从作品编辑页进入章节列表，可以新增、修改和删除章节。章节需要章节编号、标题和正文；正文可直接编辑或上传章节文件，同一请求不能同时使用两种正文来源。章节状态支持 `draft` 和 `published`，发布时设置发布时间，草稿不会出现在公开章节列表。

作者只能操作自己作品的章节，编辑和管理员可操作全部作品章节；控制器还会确认章节确实属于当前作品，跨作品提交会被拒绝。

## 7. 管理员后台

管理员从 `/admin` 进入仪表盘。`user`、`author`、`editor` 和未知角色即使登录，也不能访问管理员页面或对应 API。

### 7.1 仪表盘与内容管理

- 仪表盘显示用户、作品、章节和待审核投稿数量，并列出近期投稿和作品。
- 分类页可以创建、修改、停用或删除分类；分类 slug 必须符合格式且保持唯一。
- 作品页可以创建、修改、删除全部作品，设置作者、标题、简介、封面、分类和发布状态。
- 章节页可以管理任意作品的章节。
- 投稿审核页支持按状态筛选、预览安全渲染后的稿件、批准或拒绝。
- 审计日志页集中查看投稿相关审核和操作记录。

### 7.2 站点设置与邮件测试

`/admin/settings` 可管理：

- 邮箱验证开关、站点名称、站点副标题和联系邮箱。
- 主题色以及排行、新作、评论展示配置字段；当前 `allow_comments` 字段存在于设置接口，但没有被评分控制器作为评论权限开关执行。
- 环境配置中的应用、数据库、Session、缓存、队列、文件系统、邮件和二次验证参数。

设置变更会经过字段白名单和类型校验。SMTP 测试从后台填写收件地址后发送测试邮件；失败时应检查 `MAIL_MAILER`、主机、端口、账号、密码、发件地址和服务器网络，不能把测试成功误认为所有验证邮件都已送达。

## 8. 语言、主题、时区与阅读偏好

### 8.1 语言

页面顶部语言切换器使用 `LanguageController`。选择语言后：

- 当前语言写入 Session。
- 写入 `yuejing_locale` Cookie，登录用户同时保存 `preferred_locale`。
- 页面按 locale 输出 HTML `lang` 和文字方向 `dir`。
- 站点支持多语言及区域别名；部分语言使用 RTL 排版。

若未手动指定语言且启用了浏览器检测，系统会参考 `Accept-Language`。无效 locale 会被拒绝，不会被当作任意路径跳转。切换后若看到旧语言，刷新页面并检查 Cookie；当前实现会对切换响应设置禁止缓存的响应头。

### 8.2 全局主题

全局主题由 `ThemeToggle.vue` 和 `useTheme.js` 管理，支持：

- `light`：浅色主题。
- `dark`：深色主题。
- `eye-care`：护眼主题。
- `system`：跟随系统偏好。

主题选择保存在浏览器 `localStorage` 的 `yuejing-theme` 中。全局主题与章节阅读页的夜间开关独立存在。

### 8.3 时区

前端会通过浏览器推断 IANA 时区并提交 `/language/timezone`。合法时区会写入 Session、`yuejing_timezone` Cookie，并在登录状态下保存到用户账户。首次自动推断时，系统可能根据时区选择匹配语言；用户手动选过语言后，手动语言优先。

## 9. 常见使用流程

### 9.1 新读者开始阅读

1. 打开首页或作品库。
2. 使用关键词、分类或热门/最新排序找到作品。
3. 打开作品详情，选择已发布章节。
4. 登录后重新打开章节，即可获得阅读记录和继续阅读入口。

### 9.2 收藏并评分

1. 登录并打开已发布作品详情。
2. 点击收藏，之后在个人中心的收藏页查看。
3. 填写总评分、可选评分文字和四项维度评分并提交。
4. 需要修改评分时，先撤回旧评分，再提交新评分。

### 9.3 作者投稿到发布

1. 以 `author` 身份注册或使用已有作者账号登录。
2. 确认邮箱已验证（当站点启用邮箱验证时）。
3. 从个人中心打开投稿区域，填写标题、首次投稿简介、正文和封面。
4. 在编辑器输入正文或上传文件，二选一后提交。
5. 在投稿历史查看 `pending`、`approved` 或 `rejected`。
6. 审核通过后，从作者作品列表进入作品和章节管理；审核拒绝时根据备注修改内容后重新投稿。

### 9.4 管理员审核投稿

1. 以 `admin` 登录并打开 `/admin/submissions`。
2. 查看投稿人、分类、稿件预览和状态，必要时按状态筛选。
3. 选择批准或拒绝并填写审核备注。
4. 批准后检查自动创建的作品和第一章，再按需要调整状态、分类或后续章节。
5. 在审计日志确认审核动作和关联投稿记录。

### 9.5 发起私信或群聊

1. 登录后打开个人中心的社交分组。
2. 私信：搜索其他用户，选中后发送不超过 5000 字的消息。
3. 群组：创建群组并选择成员；成员进入群组后即可发言。
4. 需要维护成员时，由群组 `owner` 或 `admin` 添加，只有 `owner` 移除。
5. 遇到未读数没有清零时，打开对应会话并执行已读操作后刷新列表。

## 10. 权限边界与安全注意事项

- 认证页面和消息 API 使用 Session Cookie；浏览器端 POST、PUT、PATCH、DELETE 请求必须携带 CSRF Token。
- 前端不显示入口不等于授权；所有关键边界同时由路由中间件、控制器 `abort_unless` 或模型关联校验保护。
- 普通 `user` 不能通过请求参数把自己变为 `editor` 或 `admin`。
- 作者不能通过替换作品或章节 ID 操作其他作者内容；编辑和管理员才有全量作品管理能力。
- 投稿正文、章节正文和评分附带文字都经过字段长度、格式和必填校验；Markdown 预览由服务端渲染器输出，不能把原始 HTML 或脚本当作可信内容。
- 私信只能读取会话成员内容；群组只能读取、发送和标记自己所属群组的内容。
- SSE 流同样需要登录，并再次检查会话成员或群组成员身份。
- 评分、收藏和阅读记录只面向已发布作品；公开读者不能用这些入口操作草稿或归档内容。
- 语言切换只接受配置中的 locale，并限制重定向回本站点，不能借此构造外部跳转。
- 管理员环境配置包含邮件、数据库和二次验证参数，生产环境应限制管理员账号并避免在日志、截图或提交中暴露凭据。

## 11. 故障入口与排查顺序

| 现象或状态 | 常见原因 | 处理入口 |
| --- | --- | --- |
| 401 或跳转 `/login` | 需要登录或 Session 已失效 | 重新登录，检查 Cookie、Session 配置和系统时间 |
| 403 | 角色不足、邮箱未验证、不是作品所有者、不是会话/群组成员 | 确认账号角色、邮箱状态、资源归属和群组成员身份；不要反复修改 URL |
| 404 | 作品/章节不存在、未发布或章节绑定不匹配 | 从作品库和作品详情重新进入；确认状态为 `published` |
| 登录凭据错误或频繁失败 | 邮箱/密码不匹配或登录限流 | 使用密码重置入口，等待限流窗口结束 |
| 登录后出现二次验证挑战 | 账号已启用 TOTP | 使用身份验证器或恢复码；挑战过期/锁定则重新登录 |
| 注册后无法投稿或进入后台 | 邮箱验证要求已开启但账号未验证 | 从账户中重新发送验证邮件并完成签名链接验证 |
| 邮箱验证邮件未到 | SMTP 未配置、地址错误、邮件被拦截或发送异常 | 重发验证邮件；管理员到 `/admin/settings` 做 SMTP 测试并检查应用日志 |
| 投稿返回 422 | 缺少正文、首次投稿缺简介、缺封面、格式错误或同时提交两种正文来源 | 按表单错误逐项修正，正文输入和文件上传二选一 |
| 评分返回冲突错误 | 已存在有效评分 | 先撤回评分，再提交新评分 |
| 私信/群组返回 403 | 会话或群组不属于当前用户 | 从自己的会话/群组列表进入，不要使用他人的 ID |
| 群组添加/移除成员失败 | 群组角色不足或试图移除 owner | 添加由 owner/admin 执行；移除只由 owner 执行且不能移除创建者 |
| SMTP 测试失败 | 邮件驱动、主机、端口、凭据或发件地址错误 | 管理员检查站点设置与环境配置，查看 `storage/logs/laravel.log` |
| 语言切换后页面仍为旧语言 | 浏览器或 CDN 缓存、Cookie 未更新 | 检查 `yuejing_locale`，刷新带新请求的页面；确认 locale 在支持列表中 |
| 图片、封面或前端交互异常 | `public/storage`、Vite 构建产物或文件权限不完整 | 按部署文档检查 `storage:link`、`public/build/manifest.json`、PHP-FPM/Nginx 日志 |

联系页面是 `/contact`。页面显示的联系邮箱来自管理员站点设置或邮件配置；涉及账号、投稿或私信问题时，应同时提供账号邮箱、资源名称、发生时间和页面状态码，避免提交密码、TOTP 密钥、恢复码或 SMTP 凭据。
